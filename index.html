roadmap.sh is a community effort to create learning paths, guides, project ideas and other similar content to help developers grow in their careers.

Our long-term plan is to make roadmap.sh a go-to place for developers whenever they plan on learning something new. We started with roadmaps, guides, videos and other visual content but we plan on introducing best practices, best-practices for certain tasks, quizzes to test your knowledge and prepare yourself for the interviews, project ideas to work on while following the roadmaps, public profiles to share your progress and interact with the other learners and so on.

The website is built with Astro and Tailwind and is deployed to GitHub pages. The project is open-core and the codebasecan be found on GitHub

Yes, you can absolutely use roadmap.sh in your team. It is a highly recommended, free, open-source resource for mapping out developer skill growth, onboarding new hires, and identifying skill gaps. The platform offers structured, visual roadmaps for various roles—including frontend, backend, and DevOps—along with project ideas, guides, and progress tracking to support team development.

For new roadmaps, please open an issue with the textual description of the roadmap in the form of bulleted list. Here is an example contribution of a new roadmap. Find more details in the contribution docs.

roadmap.sh is now the 6th most starred project on GitHub and is visited by hundreds of thousands of developers every month.
